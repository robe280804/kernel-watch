# Components — file-by-file reference

Every source file, what it does, and its key types/functions.

---

## `main.go`

**Role:** Entry point and orchestrator.

Flow:
1. Configures structured JSON logging via `slog` at `Info` level.
2. `config.Load()` — fails fast on invalid configuration.
3. Builds components: `container.New(5*time.Minute)` (mapper), `alerter.New(cfg)`,
   `detector.New()`, `collector.New(cfg, mapper)`.
4. `coll.Start()` returns the event channel; on error it prints the hint
   *"run as root (or with CAP_SYS_ADMIN) on Linux kernel 5.15+"* and exits.
5. Installs a `signal.NotifyContext` for SIGINT/SIGTERM.
6. **Event loop:** for each event, `detect.Check(event)`; if it returns an alert,
   `alert.Send(a)`. Logs `processed`/`alerted` stats every 10 000 events. Exits
   cleanly when the channel closes or a signal arrives.

> The eBPF `//go:generate` directive lives in `internal/collector/gen.go` (not here),
> so the generated scaffolding lands in the `collector` package.

## `ebpf/tracer.c`

**Role:** Kernel-space program. Compiled by `go generate` (bpf2go) — note the
`//go:build ignore` tag so the Go toolchain skips it during normal builds.

- Defines `struct event` (must match `RawEvent`; see
  [architecture.md](architecture.md)).
- Declares the ring-buffer map `events` (`BPF_MAP_TYPE_RINGBUF`, `max_entries =
  1<<24` = 16 MB).
- `fill_common(e, type)` — populates PID/PPID/UID/type/timestamp/`comm` for every
  event using `bpf_get_current_pid_tgid`, `bpf_get_current_uid_gid`,
  `bpf_ktime_get_ns`, `bpf_get_current_comm`.
- Four tracepoint programs:
  - `trace_execve` (`sys_enter_execve`) — reads `args[0]` (filename) with
    `bpf_probe_read_user_str`.
  - `trace_openat` (`sys_enter_openat`) — reads `args[1]` (path).
  - `trace_connect` (`sys_enter_connect`) — reads the `sockaddr_in` from `args[1]`;
    records port and address only for `AF_INET` (IPv4, family == 2).
  - `trace_clone` (`sys_enter_clone`) — common fields only.
- `LICENSE[] = "GPL"` — required to use GPL-only BPF helpers.

> The in-kernel comment for `openat` says filtering is deferred to user space — the
> kernel records all opens, and `isNoisyPath` filters them in `collector.go`.

## `internal/config/config.go`

**Role:** Load and validate all runtime configuration from `KW_*` environment
variables. There are no config files.

- `Config` struct — every field maps 1:1 to a `KW_*` variable (see
  [configuration.md](configuration.md)).
- `Load()` — applies defaults, then overrides from env, then `validate()`.
- `validate()` — checks `AlertMinSeverity ∈ {low,medium,high,critical}`, `APIPort`
  in 1–65535, and that a URL is present when webhook/Slack is enabled.
- `IsMonitored(name)` — blacklist wins; if a whitelist is set only its members are
  monitored; otherwise everything not blacklisted is monitored. Case-insensitive.
- `DSN()` — builds the PostgreSQL connection string (used once persistence lands).
- Helpers: `envOr`, `envInt`, `envBool`, `splitCSV`.

> Note: a few fields are only ever set from defaults/env in the current code; the
> ring-buffer size is read into `cfg.EBPFRingbufSize` but not applied at eBPF load
> time (see [roadmap.md](roadmap.md)).

## `internal/collector/collector.go`

**Role:** The bridge between kernel and user space.

- `RawEvent` — mirrors the C struct exactly (layout-critical).
- `Event` — the enriched, parsed event (strings, `net.IP`, `time.Time`, plus
  `*container.Info`). `TypeName()` maps the type byte to `execve/open/connect/
  clone/unknown`.
- `Collector` — holds config, mapper, generated `tracerObjects`, attached `links`,
  and the `ringbuf.Reader`.
- `New(cfg, mapper)` — constructor; does not load anything yet.
- `Start()` —
  1. `rlimit.RemoveMemlock()` (eBPF maps need the lock removed),
  2. `loadTracerObjects(...)` (generated by bpf2go),
  3. attaches the four tracepoints (`link.Tracepoint`),
  4. opens the ring-buffer reader,
  5. launches `readLoop` and returns the buffered channel (cap 1000).
- `Stop()` / `cleanup()` — close the reader, detach links, close objects.
- `readLoop(out)` — read → decode → `parseEvent` → enrich via mapper → filter
  (monitored? noisy path?) → non-blocking send. `defer close(out)` signals
  shutdown to `main`.
- Helpers: `cstring` (null-terminated bytes → string), `isNoisyPath`.

> Quirk: `loadTracerObjects` is called with a `CollectionOptions` whose comment
> says "override ring buffer size from config", but no size override is actually
> performed (`PinPath: ""` only).

## `internal/container/mapper.go`

**Role:** Resolve a PID to Docker container metadata, with caching.

- `Info` — `ID` (64-char), `ShortID` (12), `Name`, `ImageName`, `ResolvedAt`.
- `Mapper` — PID→`*Info` cache (`nil` means host process) guarded by `RWMutex`,
  with a TTL and a background `evictLoop` (every 30 s).
- `Resolve(pid)` — cache lookup, else `resolveFromProc`.
- `resolveFromProc(pid)` — reads `/proc/<pid>/cgroup`; a missing file (exited
  process) is treated as "host", not an error.
- `extractContainerID(cgroup)` — parses both cgroups **v1** (`/docker/<64hex>`) and
  **v2** (`docker-<64hex>.scope`), validating 64 hex chars.
- `dockerInspect(id)` — **stub**: always returns `not implemented`; so `Name`
  falls back to the 12-char short ID and `ImageName` stays empty.
- `Invalidate(pid)` — drop a PID from the cache.
- `ParseDockerList(data)` — parses `GET /containers/json` into a map keyed by full
  and short ID. Helper for future Docker-API enrichment; **not wired in yet**.
- `isHex` — hex validation helper.

## `internal/detector/detector.go`

**Role:** Rule-based detection engine.

- `Rule` — `func(collector.Event) *alerter.Alert`.
- `Detector` — holds the ordered ruleset built in `New()`.
- `Check(event)` — returns `nil` for host events; otherwise runs rules in order and
  returns the **first** alert, stamping container/process/syscall/timestamp fields.
- Six rules (full detail in [detection-rules.md](detection-rules.md)):
  `ruleShellInContainer`, `rulePrivilegedProcessInContainer`,
  `ruleSensitiveFileAccess`, `ruleUnexpectedNetworkTool`,
  `rulePackageManagerInContainer`, `ruleCredentialFileAccess`.

> Behaviour to know: only the first matching rule fires per event, and matching is
> by exact process name (`comm`, lower-cased) for exec rules and by path prefix for
> file rules.

## `internal/alerter/alerter.go`

**Role:** Format and dispatch alerts; enforce severity threshold and rate limits.

- `Severity` constants + `severityRank` for threshold comparisons.
- `Alert` — the JSON-serialisable alert payload (id, server, timestamp, severity,
  container id/name, image, syscall, pid, process, reason, details, MITRE TTP/tactic).
- `Alerter` — config, log file handle, HTTP client (5 s timeout), and per-container
  rate-limit state.
- `New(cfg)` — opens/creates the log file and its directory when logging is enabled.
- `Send(alert)` — severity filter → rate limit → stamp server/timestamp → dispatch
  to log (sync) + webhook + Slack (each in a goroutine).
- `writeLog` — appends newline-delimited JSON to the file **and** emits a structured
  `slog.Warn`.
- `sendWebhook` — POSTs JSON; if `KW_WEBHOOK_SECRET` is set, adds an
  `X-KernelWatch-Signature: sha256=<hmac>` header (HMAC-SHA256 of the body).
- `sendSlack` — builds a Slack Block-Kit message with a severity emoji and the
  MITRE line.
- `isRateLimited(containerID)` — sliding window: evicts timestamps older than the
  window, blocks if the count reaches `AlertMaxRate`, else records "now".

> Note: this file now imports the real `strings` package. (An earlier version had a
> package-level `strings` shim shadowing the standard library — removed during
> stabilization.)

## Build & deploy files

- **`go.mod`** — module `kernelwatch`, Go 1.22, single require:
  `github.com/cilium/ebpf v0.15.0`. (No `go.sum` committed yet; the Dockerfile
  `COPY go.sum` and `go mod download` will need it.)
- **`Dockerfile`** — multi-stage: builder (`golang:1.22-bookworm` + clang/LLVM/
  libelf/headers, runs `go generate`, builds a stripped static binary) → runtime
  (`debian:bookworm-slim` + libelf, non-root user UID 1000). See
  [deployment.md](deployment.md).
- **`docker-compose.yml`** — the `kernelwatch` daemon (host network, scoped
  capabilities, host mounts, healthcheck) plus a `timescaledb` service
  (`./migrations` auto-run on first start, bound to localhost only).
- **`.env.example`** — documented template of every `KW_*` variable.
- **`README.md`** — top-level project readme.
- **`docs/`** — this documentation (`en/` and `it/`).
